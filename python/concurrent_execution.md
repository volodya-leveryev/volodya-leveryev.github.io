# Теория

Процесс - виртуальное адресное пространство в котором присутствует хотя бы один поток команд.
Процесс создается ОС. Процесс имеет уникальный номер и может стать владельцем ресурсов:
- открытые файлы
- сетевые соединения
- механизмы межпроцессного обмена данными
- и т.д.
Процессы надежно изолированы друг от друга. Процесс может быть в любой момент уничтожен ОС.

Поток - Последовательность инструкций процессора, которая работает в рамках процесса.

GIL - механизм в интерпретатора Python который запрещает одновременное исполнение нескольких операторов Python.

Ограничения скорости работы приложений:
- количество вычислений - CPU bound
- ввод-вывод данных (включая сетевой обмен или ожидание ответа от пользователя) - I/O bound

Часто можно ускорить работы приложений написав более качественный алгоритм.

Как ускорить вычислительно сложную задачу на Python:
- найти более быстрый процессор
- написать лучший алгоритм
- использовать multiprocessing
- использовать библиотеки на другом языке программирования
